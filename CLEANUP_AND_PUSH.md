# Быстрая очистка контролов и пуш изменений

Коротко — что я сделал/сделаю и как быстро зафиксировать изменения в репозитории.

1) Цель
- Убрать возможность управления воспроизведением из интерфейса слушателя (`/user?room_id=...`).
- Оставить у слушателя только действия: Connect / Disconnect к потоку.
- Перенести всё управление (Start/Stop/Pause/Next) только в Live (админский) интерфейс.

2) Изменения (порядок и места)
- Сервер: `backend/app/websocket/handlers.py` и `backend/app/stream/router.py` — отклонять WS/HTTP команды управления от не‑админов.
- Фронтенд (user view): `frontend/src/pages/RoomPage.jsx` + legacy `frontend/public/static/js/room/*` — заменить/скрыть кнопки воспроизведения, присвоить им функции Connect/Disconnect.

3) Быстрый чек (локально)
```bash
# из корня репозитория
git status
git add CLEANUP_AND_PUSH.md
git commit -m "chore: add cleanup instructions for locking room controls to admin"
git remote get-url origin
git push origin HEAD
```

4) Примечания по пушу
- Пуш требует настроенного remote `origin` и корректных прав (SSH или HTTPS с токеном).
- Если push не проходит — пришлю инструкцию в этом файле с командами для вашего окружения (SSH/HTTPS).

5) Тесты после пуша
- Откройте `/user?room_id=2` в режиме обычного пользователя: кнопки Play/Pause/Next должны быть скрыты или неактивны; видна только кнопка Connect/Disconnect и индикатор состояния.
- Откройте `/live` как админ: все кнопки управления должны работать.

Если подтверждаете — я сделаю коммит и попробую запушить в ваш origin (https://github.com/lilyaka1/Omni-Player). Если пуш не получится автоматически, дам точные команды для ручного выполнения.
