# Описание диаграмм OmniPlayer

## UseCase-диаграмма

UseCase-диаграмма фиксирует акторов и их действия в системе OmniPlayer. 
В системе определены три типа акторов: **Гость**, **Пользователь** и **Администратор**.

### Гость (неавторизованный пользователь)
- **Регистрация** — создание учётной записи с email и паролем
- **Вход** — авторизация в системе
- **Просмотр публичных комнат** — просмотр списка доступных комнат без авторизации

### Пользователь (авторизованный)
- **Создание комнаты** — создание новой комнаты с настройкой имени и пароля
- **Присоединение к комнате** — вход в комнату по ссылке или ID
- **Управление очередью** — добавление, удаление и переупорядочивание треков
- **Воспроизведение** — запуск, пауза, пропуск треков
- **Отправка сообщений** — общение в чате комнаты
- **Просмотр участников** — просмотр списка слушателей в комнате

### Администратор
- **Модерация чата** — удаление сообщений и блокировка пользователей
- **Управление комнатами** — просмотр и удаление всех комнат
- **Управление пользователями** — просмотр списка, изменение ролей
- **Просмотр аудита** — доступ к журналу действий пользователей

## UserStoryMap

UserStoryMap организована по этапам (слева направо):

### 1. Discovery (Открытие и вход)
- Вход на сайт
- Регистрация
- Вход в систему
- Просмотр списка комнат

### 2. Home/Library (Библиотека)
- Просмотр библиотеки треков
- Загрузка аудиофайла
- Поиск трека
- Просмотр профиля

### 3. Room Lifecycle (Создание и вход в комнату)
- Создание комнаты
- Настройка параметров (название, пароль, публичность)
- Присоединение к комнате
- Ввод пароля (если приватная)
- Просмотр участников

### 4. Playback (Воспроизведение)
- Добавление трека в очередь
- Удаление трека из очереди
- Изменение порядка очереди
- Запуск воспроизведения
- Пауза/Возобновление
- Пропуск трека
- Синхронизация с другими участниками

### 5. Communication (Коммуникация)
- Отправка сообщения в чат
- Просмотр истории чата
- Включение голосовых вставок (озвучка названия трека)

### 6. Admin (Администрирование)
- Модерация чата
- Блокировка пользователя
- Управление правами участников
- Просмотр журнала аудита

### Связи между этапами

Каждый этап логически связан с предыдущим:
- После **Discovery** пользователь переходит к **Library** или сразу к **Room**
- Из **Room Lifecycle** пользователь попадает в **Playback**
- Параллельно с воспроизведением работает **Communication**
- Административные функции (**Admin**) доступны через настройки комнаты

## Figma-макеты

### Главный экран комнаты (Room Page)
Содержит:
- **Плеер** — обложка, название трека, имя артиста, прогресс-бар
- **Управление** — кнопки play/pause, prev/next, громкость
- **Очередь** — список треков с возможностью перетаскивания
- **Чат** — поле ввода сообщений, история
- **Слушатели** — список участников комнаты

### Список комнат / Вход
- Форма входа (email, пароль)
- Форма регистрации
- Карточки публичных комнат
- Кнопка создания новой комнаты
